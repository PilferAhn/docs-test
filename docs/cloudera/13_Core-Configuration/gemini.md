# Core Configuration (Gemini 답변)

> 출처: Gemini 답변 원문
> Gemini는 Core Configuration을 개별 컴포넌트로 별도 다루지 않았습니다.

## Gemini의 관련 내용

Core Configuration은 Gemini 답변에서 명시적으로 언급되지 않았습니다.

### 간접 관련 맥락
Gemini 답변에서 설명한 두 플랫폼의 구조적 차이가 Core Configuration의 위치를 잘 설명합니다.

- **Cloudera**: 각 서비스가 독립적으로 운영 → 공통 설정을 별도 서비스로 관리 필요
- **Databricks**: 통합 플랫폼 → 공통 설정이 플랫폼 안에 흡수됨

이는 Gemini가 설명한 "Cloudera는 도구의 다양성, Databricks는 엔진의 통일성"이라는 핵심 차이와 일치합니다.
